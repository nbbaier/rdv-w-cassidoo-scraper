# RSS-only ingestion

The scraper was previously a two-step pipeline: crawl Buttondown's paginated `/archive/` HTML pages to discover newsletter URLs, then fetch each issue's HTML page individually and extract the interview question. We replaced this with a single RSS request to `https://buttondown.com/cassidoo/rss?count=N`, which returns up to N items with full HTML bodies in each `<description>` — meaning one HTTP call gets either the entire backlog (`count=1000` → all ~457 issues at the time of writing) or the weekly delta (`count=20` is plenty of slack for missed cron runs). The trade-off: we depend on Buttondown's `?count=` query param continuing to behave this way — it's undocumented and could be capped or removed unilaterally, at which point the archive-crawl approach would need to be rebuilt from scratch (and won't be in HEAD anymore). We accepted that risk because the single-request shape eliminates pagination guesswork, the per-issue fetch loop, the intermediate `data.json`, and ~half the scraper code.
